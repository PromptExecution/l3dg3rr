# tax-ledger

[![CI](https://github.com/PromptExecution/l3dg3rr/actions/workflows/ci.yml/badge.svg)](https://github.com/PromptExecution/l3dg3rr/actions/workflows/ci.yml)
[![Release](https://github.com/PromptExecution/l3dg3rr/actions/workflows/release.yml/badge.svg)](https://github.com/PromptExecution/l3dg3rr/actions/workflows/release.yml)
[![Documentation](https://img.shields.io/badge/docs-github.io-blue)](https://promptexecution.github.io/l3dg3rr/)

A local-first, Excel-first personal financial document intelligence system for retroactive US expat tax preparation. Ingests raw PDF and CSV bank statements, classifies transactions with runtime-editable Rhai rules, and produces a CPA-auditable Excel workbook with Schedule-oriented outputs and full mutation history.

**Core value**: Convert raw historical financial PDFs into accountant-usable, auditable Excel tax records without sending private data to third-party SaaS.

---

## What it does

1. **Ingest** — Accepts statement files named `VENDOR--ACCOUNT--YYYY-MM--DOCTYPE` (e.g. `chase--checking--2024-01--statement.pdf`). The filename convention drives deterministic routing to the correct extraction profile.
2. **Classify** — Runs a waterfall of Rhai rule files over each transaction. Rules are runtime-editable; no recompile required. Each rule returns `{ category, confidence, review, reason }`.
3. **Audit** — Every mutation is appended to a structured audit log. The CPA workbook contains an AUDIT sheet with full mutation history.
4. **Export** — Writes a `rust_xlsxwriter` Excel workbook with Transactions, Schedule C/D/E summary sheets, Flags, and Audit tabs. A CPA can open the workbook without any special tooling.

The live docs editor supports two synchronized views for supported ` ```rhai ` workflow blocks:
- `isometric-3d` for animated constraint layout with inline SVG scene rendering and autogenerated glTF node-model fallbacks
- `mermaid-2d` for the canonical Mermaid flowchart view

Chapters include:
- Graph Data Model
- Force Layout
- Isometric Projection
- Renderer
- Slint Visualization
- Pipeline
- Validation
- Legal Verification
- Constraints
- Verification
- Workflow
- Visualization

The system is designed for an operator/agent workflow: AI performs ingestion, classification, reconciliation, and flagging; a human accountant reviews and signs off in Excel.

## Rhai Tax Classification Rules

Twelve `.rhai` rule files live in `rules/`. Each implements `fn classify(tx)` accepting `{ tx_id, account_id, date, amount, description }` and returning `#{ category, confidence, review, reason }`. `amount` is always a string decimal; rules use `parse_float()` for numeric comparisons only.

| Rule File | Tax Code | Jurisdiction | Output Categories | Review Trigger |
|---|---|---|---|---|
| `classify_schedule_c.rhai` | IRC §162(a) | US | `SelfEmployment`, `OfficeSupplies` | Income > $5,000 or expense abs(amount) > $2,500 |
| `classify_schedule_d.rhai` | IRC §1222(1), §1222(3) | US | `CapitalGain`, `CapitalLoss` | Description contains "short" or "< 1 year" (STCG taxed as ordinary income) |
| `classify_schedule_e.rhai` | IRC §469(c)(1) | US | `RentalIncome` | Amount > $10,000; always review on negative |
| `classify_fbar.rhai` | 31 USC §5314; FinCEN Form 114 | US | `ForeignIncome` | abs(amount) > $9,000 (approaching FBAR threshold) |
| `classify_fatca.rhai` | IRC §6038D; Form 8938 | US | `ForeignIncome` | abs(amount) > $25,000 (Form 8938 threshold) |
| `classify_crypto_trading.rhai` | IRS Notice 2014-21; Rev. Proc. 2024-28 | US | `Transfer` (buy), `CryptoGain` / `CapitalLoss` (sell) | Always on sell/exchange (must compute basis and holding period) |
| `classify_crypto_staking.rhai` | Rev. Rul. 2023-14; IRC §61(a)(1) | US | `CryptoIncome` | Always (must determine FMV at receipt) |
| `classify_au_gst.rhai` | GST Act 1999 s.11-5 | AU | `AuGst` (expense), `ForeignIncome` (income) | Always on positive AU income (US expat foreign income obligation) |
| `classify_au_cgt.rhai` | ITAA 1997 s.115-A | AU | `AuCgt` | Always (50% CGT discount eligibility requires acquisition date verification) |
| `classify_foreign_income.rhai` | — | US | `ForeignIncome` | — |
| `classify_self_employment.rhai` | — | US | `SelfEmployment` | — |
| `classify_fallback.rhai` | — | — | `Unclassified` | — |
| `classify_document_shape.rhai` | — | — | (shape routing, not a transaction classifier) | — |

Rules return `Unclassified` with `confidence: 0.0` when no signal matches, allowing the engine to continue to the next rule. FBAR and FATCA rules detect individual transaction signals only — aggregate balance across all foreign accounts must be computed by the Rust pipeline.

The rule ontology is documented in `rules/ONTOLOGY.md`.

## BusinessCalendar

`crates/ledger-core/src/calendar.rs` implements deterministic tax deadline scheduling and recurring pipeline triggers.

### RecurrenceRule variants

- `Monthly { day_of_month }` — fires on the Nth day of every month
- `Annual { month, day }` — fires once per year on month/day
- `QuarterlyEstimated { day }` — fires in April, June, September, January (US estimated tax schedule; AU BAS uses same pattern)
- `EveryNDays { n }` — fires every N days
- `CronExpr(String)` — stored for forward compatibility; evaluation is a stub (returns `None`)

### Key methods

- `BusinessCalendar::from_toml_file(path)` — load from TOML manifest
- `BusinessCalendar::next_due(rule, after)` — pure date arithmetic; fully implemented
- `BusinessCalendar::upcoming(from, horizon_days)` — returns `(date, event)` pairs sorted ascending
- `BusinessCalendar::events_due_on(date)` — events firing on exactly `date`
- `BusinessCalendar::events_by_tag(tag)` — filter by tag string
- `BusinessCalendar::merge(other)` — combine US + AU calendars for multi-jurisdiction entities
- `BusinessCalendar::us_tax_defaults()` — pre-populated US events (Form 1040, FBAR, FATCA, quarterly estimates, monthly ingest/classify)
- `BusinessCalendar::au_tax_defaults()` — pre-populated AU events (BAS quarters, individual return, super contribution, CGT review)

### TOML calendar manifests

Two TOML manifests ship with the repository:

- `calendar/tax_calendar_us.toml` — 10 events covering US federal deadlines with tax code citations (IRC §6654, 31 USC §5314, IRC §6038D, Form 4868)
- `calendar/tax_calendar_au.toml` — 10 events covering AU obligations (BAS Q1–Q4, ITAA 1997 individual return, super concessional cap s.290-180, CGT discount review s.115-A)

All date arithmetic is tested in the `calendar` module test suite (17 tests).

## LedgerOperation Trait and OperationDispatcher

`crates/ledger-core/src/ledger_ops.rs` defines the composable pipeline operation interface.

### Trait

```rust
pub trait LedgerOperation: Send + Sync {
    fn id(&self) -> &str;
    fn description(&self) -> &str;
    fn is_idempotent(&self) -> bool { false }
    fn execute(&self, ctx: &OperationContext) -> Result<OperationResult, LedgerOpError>;
}
```

### OperationKind enum (carried in ScheduledEvent)

```rust
pub enum OperationKind {
    IngestStatement { source_glob: String },
    ClassifyTransactions { rule_dir: String },
    ReconcileAccount { account_id: String },
    ExportWorkbook { output_path: String },
    GenerateAuditTrail { year: i32 },
    CheckTaxDeadline { deadline_id: String },
}
```

### Six concrete operations — all return `LedgerOpError::NotImplemented`

| Struct | ID | Idempotent | Intended behavior |
|---|---|---|---|
| `IngestStatementOp` | `ingest-statement` | Yes (Blake3 dedup) | Expand glob, detect shape, call Docling sidecar, compute content-hash IDs, upsert to ledger, write `.rkyv` sidecar |
| `ClassifyTransactionsOp` | `classify-transactions` | Yes | Load `.rhai` rules via `RuleRegistry`, run waterfall over unclassified transactions, flag low-confidence |
| `ReconcileAccountOp` | `reconcile-account` | No | Match local vs Xero transactions, flag unmatched |
| `ExportWorkbookOp` | `export-workbook` | Yes (overwrite) | Write full workbook with Transactions, Schedule summaries, Flags, Audit sheets |
| `GenerateAuditTrailOp` | `generate-audit-trail` | Yes (overwrite) | Export year-scoped mutation log to structured JSON/XLSX |
| `CheckTaxDeadlineOp` | `check-tax-deadline` | Yes | Look up deadline in `BusinessCalendar`, emit advisory if approaching |

`OperationDispatcher` collects operations via `register()` and runs them with `run_all(ctx)` or `run_by_id(id, ctx)`.

## DocumentShape Classifier

`crates/ledger-core/src/document_shape.rs` classifies the bank/vendor and statement format from a document before extraction. The classifier runs 8 sequential passes:

1. Parse `VENDOR--ACCOUNT--YYYY-MM--DOCTYPE` filename convention (+0.5 confidence on vendor match)
2. Filename keyword scan (wellsfargo, wf\_, chase, jpmc, hsbc, anz, commbank, cba, westpac, interactive, coinbase, kraken, bankofamerica, boa) (+0.4)
3. Content-based vendor detection (Chase CSV header, Wells Fargo text, BSB code, IBAN/BIC/SWIFT, CommBank/Westpac text, AU/USD currency clues) (+0.2–0.5)
4. Date format detection (ISO `%Y-%m-%d`, US slash `%m/%d/%Y`, AU slash `%d/%m/%Y`)
5. CSV column map inference (maps source headers to canonical: `date`, `amount`, `description`, `balance`, `category`)
6. Statement format fallback (`csv_generic`, `xlsx_native`, `pdf_tabular`, `unknown`)
7. AU vendor → AUD currency reconciliation (ANZ, CommBank, WestpacAu always produce AUD)
8. Generic CSV fallback for unrecognized CSV files

**12 `StatementVendor` variants**: `WellsFargo`, `Chase`, `BankOfAmerica`, `Hsbc`, `Anz`, `Commbank`, `WestpacAu`, `Interactive`, `Coinbase`, `Kraken`, `Generic`, `Unknown`

Each vendor carries an implied jurisdiction (`US`, `AU`, or `UK` for HSBC) and a stable slug used in filenames and rule files.

## RuleRegistry

`crates/ledger-core/src/rule_registry.rs` defines the interface for multi-rule orchestration. All three core methods are stubs (`unimplemented!()`).

```rust
pub struct RuleRegistry {
    rule_paths: Vec<PathBuf>,
    candidates: Vec<Option<ReqIfCandidate>>,
}

impl RuleRegistry {
    // STUB: scan rules_dir for .rhai files; optionally load .reqif.json sidecars
    pub fn load_from_dir(rules_dir: &Path) -> Result<Self, RuleRegistryError>

    // STUB: keyword-match rule selection; returns all rules when no match found
    pub fn select_rules_deterministic(&self, tx: &SampleTransaction) -> Vec<PathBuf>

    // STUB: first-non-Unclassified-wins waterfall over selected rules
    pub fn classify_waterfall(
        &self,
        engine: &mut ClassificationEngine,
        tx: &SampleTransaction,
    ) -> Result<ClassificationOutcome, ClassificationError>
}
```

The `SemanticRuleSelector` trait (requiring embedding infrastructure) is also defined but unimplemented:
- `select_rules_semantic(tx, top_k)` — vector similarity search over `ReqIfCandidate` embeddings
- `build_embedding_index()` — builds HNSW index from loaded candidates

Two mirror types for the Python `reqif-opa-mcp` sidecar's JSON output are defined: `ReqIfCandidate` and `DocumentChunk`.

## mdbook-rhai-mermaid Preprocessor

`crates/mdbook-rhai-mermaid/` is a standalone mdBook preprocessor binary that converts `\`\`\`rhai` diagram DSL blocks into injected `\`\`\`mermaid` flowchart blocks at build time.

### DSL syntax (three forms)

```rhai
// Pipeline step
fn source_node() -> target_node

// Conditional branch
if confidence > 0.85 -> commit_workbook

// Match arm
match result.disposition => Disposition::Recoverable -> repair_and_retry
```

### Processing pipeline

The preprocessor reads `[PreprocessorContext, Book]` from stdin (mdBook protocol), walks every chapter and sub-chapter, finds each `\`\`\`rhai` fence, parses the body into a `Graph` (nodes + edges), emits a `flowchart TD` Mermaid block immediately after the rhai block, then writes the modified book to stdout.

Threshold chains using `>` or `<` operators are automatically chained with `true`/`false` edges (e.g. `confidence > 0.85` and `confidence > 0.60` produce a two-node diamond chain). Match arms with the same `expr` are grouped into a single match node.

The parser strips `//` inline comments and silently skips unrecognized lines — a partial parse always succeeds.

### Key files

- `crates/mdbook-rhai-mermaid/src/main.rs` — mdBook preprocessor entrypoint, fence detection, Mermaid injection
- `crates/mdbook-rhai-mermaid/src/parser.rs` — graph AST, three DSL forms, threshold chain builder
- `crates/mdbook-rhai-mermaid/src/emitter.rs` — `Graph` → `flowchart TD` Mermaid string
